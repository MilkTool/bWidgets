Only creates widget in-place, layout still needs to be created by the caller. Otherwise, we'd have to do perfect forwarding of parameters to multiple constructors. This can be done (e.g. see https://cpptruths.blogspot.com/2012/06/perfect-forwarding-of-parameter-groups.html) but in the end, just letting the caller create the layout is less verbose and easier to understand.

Add child node for a widget created in-place. Simplifies creating and appending widgets. The arguments __args are forwarded to the widget constructor; a new node for it is added as child to node.

Without emplaceWidget():

using bWidgets::bwScreenGraph;
auto& widget = static_cast<bwLabel&>(Builder::addWidget<bwLabel>(screen_graph, bwPtr_new<bwLabel>("Foo", 0, 10)));
widget.foo();
// ...

With emplaceWidget():

using bWidgets::bwScreenGraph;
auto& widget = Builder::emplaceWidget<bwLabel>(screen_graph, "Foo", 0, 10);
widget.foo();
// ...
