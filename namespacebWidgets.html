Detailed Description

Simple type-trait mechanism to get the right bwStyleProperty::PropType from the data-type of the property (bool, int, bwColor, ...) and vise versa.

template<typename _PointerType >

Alias for std::unique_ptr().

With this,std::unique_ptr() follows our naming conventions. In future this may also be useful in case we add our own unique-pointer implementation.

We currently don't need a std::shared_ptr() version of this, and would like to avoid it.

Own implementation of std::make_unique() for C++11.

std::make_unique() is only introduced in C++14, this is an implementation that matches its behavior. It also follows bWidgets' naming conventions.

E.g. this:

bwPointer<SomeType> type = bwPointer<SomeType>(new SomeType(some_arg));

shortens to:

bwPointer<SomeType> type = bwPointer_new<SomeType>(some_arg);

Since the type is clear from passing it to bwPointer_new() as template parameter, it's fine to use auto too:

auto type = bwPointer_new<SomeType>(some_arg);
